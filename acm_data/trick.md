29. 求最小环长度和数量（$n\leq500$）

    考虑用 Floyd 求解，我们枚举环上编号最大的那个点 $k$ 去把环分类。

    根据当枚举到 $k$ 的时候 $g[i][j]$ 就表示 $i->j$ 的路径中，中间结点的编号范围为 $[1,k]$。 

    我们只枚举到环中编号最大的顶点的时候统计，且我们枚举 $(k,i)$ 这条边，使得他与 $g[i][k]$ 拼起来形成一个环。

    ```c++
    const int N = 505, mod = 998244353;
    LL f[N][N], g[N][N], cnt[N][N]; //cnt[i][j] 表示从i->j的最短路有几条
    int n, m;
    read(n), read(m);
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            cnt[i][j] = 0;
            if (i == j) g[i][j] = 0;
            else g[i][j] = 1e15;
        }
    }
    while (m--) {
        int u, v, w;
        read(u), read(v), read(w);
        g[u][v] = min(g[u][v], (LL)w);
        cnt[u][v] = 1;
    }
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            f[i][j] = g[i][j];
        }
    }
    
    for (int k = 1; k <= n; k++) {
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                if (g[i][j] > g[i][k] + g[k][j]) {
                    g[i][j] = g[i][k] + g[k][j];
                    cnt[i][j] = cnt[i][k] * cnt[k][j] % mod;
                } else if (g[i][j] == g[i][k] + g[k][j]) {
                    cnt[i][j] = (cnt[i][j] + cnt[i][k] * cnt[k][j] % mod) % mod;
                }
            }
        }
        for (int i = 1; i < k; i++) {
            if (res > f[k][i] + g[i][k]) {
                res = f[k][i] + g[i][k];
                sum = cnt[i][k];
            } else if (res == f[k][i] + g[i][k]) {
                sum = (sum + cnt[i][k]) % mod;
            }
        }
        
    }
    ```

30. 给定你 $n$ 个数，你可以选择任意一个子集设 $\sum_{i\in S} a_i$ ，问最小不能被凑出的正整数是啥？

    我们考虑先对 $n$ 个数排序，那么我们记 $x$，表示现在我们 $[0,x]$ 的数都可以被凑出，那么对于 $ai$

    若 $a_i>x+1$，说明 $x + 1$ 是不能被凑出来的最小非负整数，因为此时前面所有数加起来都 $<x+1$，且 $a_i>x+1$，说明后面的

    数都要 $>x+1$，故我们无论选什么子集都不可能凑出 $x+1$。 

    否则 $a_i\leq x+1$，此时可以把 $a_i$ 加入 $x$，表示我们用前 $i$ 个数，能凑出 $[1,x+a_i]$ 的数。

    同时我们得到一个 trick：

    对于一个集合 $S$，其和为 $x$，且这个集合中选任意一个子集都可以凑出 $[0,x]$ 的所有数，若要加入新的数 $a$ 加入到集合

    当且仅当，$a\leq x+1$。

33. 树的各种性质：

    - 树的直径不是唯一的。

    - 树直径的两个端点必然是树的两个叶子节点。

    - 如果树的直径大小为 $D$，则有：

      若 $D$ 为偶数，则所有直径必然汇聚在直径的中点处。

      若 $D$ 为奇数，则所有直径必然汇聚于中间边 $(u,v)$ 。

    - 某个点是树的重心当且仅当它的最大子树大小不大于整棵树大小的一半。

    - 树中所有点到某个点的**距离和**中，到重心的距离和是最小的；如果有两个重心，那么到它们的距离和一样。反过来，距离和最小的点一定是重心。

    - 树**至多有两个**重心。如果树有两个重心，那么它们**相邻**。此时树一定有**偶数**个节点，且可以被划分为两个大小相等的分支，每个分支各自包含一个重心。

    - 往树上增加或减少一个**叶子**，如果原节点数是奇数，那么重心可能**增加**一个，原重心**仍是重心**；如果原节点数是偶数，重心可能**减少**一个，另一个重心**仍是重心**。

    - 把两棵树通过一条边相连得到一棵新的树，则新的重心在**较大**的一棵树一侧的**连接点**与**原重心**之间的**简单路径**上。如果两棵树大小一样，则重心就是两个连接点。

    ```c++
    找重心
    int n, sz[MAXN], mss[MAXN]; // n：总结点数（请从外部传入），sz：树的大小，mss：最大子树大小
    vector<int> ctr; // 重心
    void dfs(int p, int fa = 0) // 找重心
    {
        sz[p] = 1, mss[p] = 0;
        for (auto [to, w] : edges[p])
            if (to != fa)
            {
                dfs(to, p);
                mss[p] = max(mss[p], sz[to]);
                sz[p] += sz[to];
            }
        mss[p] = max(mss[p], n - sz[p]);
        if (mss[p] <= n / 2) ctr.push_back(p);
    }

      ```

    - 若 $p,q$ 都是字符串的周期，那么 $\gcd(p,q)$ 也是字符串的周期。

    Border 树的性质：

    - 建树方式：$i\rightarrow nxt[i]$ 连边。

    - 每个前缀 $pre[i]$ 的所有 Border 为节点 $i$ 到根的链。
    - 有哪些前缀有长度为 $x$ 的 Border，相当于求 $x$ 的子树大小。
    - 求两个前缀的公共 Border 等价于求两个节点的 LCA。

66. 给定一棵树，求 $\sum_{i=1}^n\sum_{j=i}^nI(i,j)$，或者求 $\sum_{i=1}^n\sum_{j=i}^nP(i,j)$

    其中 $I(i,j)$ 表示树上两点 $i,j$ 路径上**边权**的最大值和最小值之差。

    其中 $P(i,j)$ 表示树上两点 $i,j$ 路径上**点权**的最大值和最小值之差。

    十分经典的 trick，**排序+并查集**。

    - 不难发现可以最大值，最小值分开来算。
    - 对于边权：以最求最大值为例，按边权从小到大排序，对于一条边 $(u,v)$ 那么所有和 $u$ 在一个集合，和 $v$ 在一个集合的点，两两之间构成的路径的最大值就是当前 $(u,v)$ 的边权，算贡献：$sz_u\times sz_v\times w_{u,v}$ 即可。
    - 对于点权：以最求最大值为例，按点权从小到大排序，然后每次遍历所有 $u$ 的邻边，若 $v$ 已经在前面被处理过了，说明此时 $v$ 所在集合的所有点都到 $u$ 所在集合所有点构成路径的最大值就是当前点 $u$ 的权值，算贡献即可。

73. 可撤销并查集：

    需要使用启发式合并，然后利用栈记录合并路径，撤销操作相当于出栈。

    ```c++
    struct DSU {
    	std::pair<int, int> stk[N];
    	int top, f[N], sz[N];
    	void init(int n) {
    		top = 0;
    		for (int i = 1; i <= n; i++) {
    			f[i] = i;
    			sz[i] = 1;
    		}
    	}
    
    	int find(int x) {
    		while (x != f[x]) x = f[x];
    		return x;
    	}
    
    	void merge(int u, int v) {
    		int pu = find(u), pv = find(v);
    		if (pu == pv) return;
    		if (sz[pu] > sz[pv]) std::swap(pu, pv);
    		f[pu] = pv;
    		sz[pv] += sz[pu];
    		stk[++top] = {pu, pv};
    	}
    
    	void roll(int t) {
    		while (top > t) {
    			auto [pu, pv] = stk[top--];
    			f[pu] = pu;
    			sz[pv] -= sz[pu];
    		}
    	}
    
    	bool same(int u, int v) {
    		return find(u) == find(v);
    	}
    };

    - $a\pm c\equiv b\pm d\pmod m$。
    - $a\times c\equiv b\times d\pmod m$。

81. 求总和与概率期望的转化

    对于一类求满足某种条件的总和很难求的问题，可以考虑转化为先求期望，定义 $E$ 为所求总和的期望， $Ans$ 为所求总和， $|S|$ 为总样本空间大小，有 $Ans = E\times |S|$

   - 例如求所有 $n!$ 中排列中，逆序对总数:
     - 直接求不好求，考虑算 $n!$ 中逆序对出现总数的期望，在考虑期望的线性性质， $E=\sum_{i<j} I_{i,j}\times P(a_i<a_j)$，其中 $I_{i,j}=[a_i<a_j]$，对于任意两个位置 $i<j$ 其 $a_i<a_j$ 的概率为 $\frac{1}{2}$， 所以 $E=\frac{n\times(n-1)}{2}\times \frac{1}{2}=\frac{n\times(n-1)}{4}$，故 $Ans=\frac{n\times(n-1)}{4}\times n!$
